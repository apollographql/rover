If your environment restricts outbound network traffic, use this page to configure your firewall allowlist for Rover. For proxy server configuration, see [Using Rover with a Proxy Server](./proxy).

## Runtime endpoints

The following table lists every hostname Rover contacts at runtime, why it contacts it, and which environment variable (if any) can override or disable the connection.

| Hostname | Purpose | Required? | Override env var |
|---|---|---|---|
| `api.apollographql.com` | GraphOS schema registry API — used by all commands that publish, check, or fetch schemas and graph configurations | Yes | `APOLLO_REGISTRY_URL` |
| `rover.apollo.dev` | Plugin and binary downloads — `rover dev` and `rover supergraph compose` download the composition plugin and router binary on first run | When using `dev` or `supergraph compose` | `APOLLO_ROVER_DOWNLOAD_HOST` |
| `rover.apollo.dev` | Anonymous usage telemetry | No | Set `APOLLO_TELEMETRY_DISABLED=true` to disable; set `APOLLO_TELEMETRY_URL` to redirect to an internal endpoint |
| `rover.apollo.dev` | Template metadata for `rover template list` | No | `APOLLO_TEMPLATES_API` |
| `github.com` / `api.github.com` | Automatic version update checks (background, all commands); template archive downloads for `rover template use` and `rover init` | No | Set `APOLLO_ROVER_SKIP_UPDATE=true` to disable update checks |
| `codeload.github.com` | GitHub's download CDN — `api.github.com` redirects template archive requests here; contacted automatically when `rover template use` or `rover init` downloads a template | When using those commands | — |
| `specs.apollo.dev` | Federation spec URLs embedded in schema directives — these are identifiers in your SDL, not fetched by Rover at runtime | No | — |

<Note>

Only `api.apollographql.com` is required for core registry operations. All other hostnames are either optional or only contacted when you use specific commands.

</Note>

## Plugin downloads

`rover dev` and `rover supergraph compose` silently download binaries from `rover.apollo.dev` on first run in a given environment. In air-gapped or locked-down environments, these downloads will fail without a clear error message.

<Caution>

If Rover cannot reach `rover.apollo.dev`, commands that require the composition plugin or router binary will fail. Pre-download the required plugins on a reachable machine and transfer them to the restricted environment, or ensure `rover.apollo.dev` is included in your allowlist.

</Caution>

## Environment variable reference

The following environment variables control which endpoints Rover contacts. For a full reference of all Rover environment variables, see [Configuring Rover](./configuring).

| Environment variable | Effect |
|---|---|
| `APOLLO_REGISTRY_URL` | Override the GraphOS API endpoint (default: `https://api.apollographql.com/graphql`) |
| `APOLLO_ROVER_DOWNLOAD_HOST` | Override the host used for plugin and binary downloads (default: `https://rover.apollo.dev`) |
| `APOLLO_TELEMETRY_URL` | Override the telemetry reporting endpoint (default: `https://rover.apollo.dev/telemetry`) |
| `APOLLO_TELEMETRY_DISABLED` | Set to `true` to disable telemetry entirely |
| `APOLLO_TEMPLATES_API` | Override the templates metadata endpoint (default: `https://rover.apollo.dev/templates`) |
| `APOLLO_ROVER_SKIP_UPDATE` | Set to `true` to disable automatic version update checks against GitHub |
