If your environment restricts outbound network traffic, use the table below to configure your firewall allowlist. For proxy server configuration, see [proxy server configuration](./proxy).

## Runtime endpoints

The following table lists every hostname Rover contacts at runtime, its purpose, whether it's required, and the environment variable (if any) that overrides or disables the connection:

| Hostname | Purpose | Required? | Override env var |
|---|---|---|---|
| `api.apollographql.com` | GraphOS schema registry API. All commands that publish, check, or fetch schemas and graph configurations use this endpoint. | Yes | `APOLLO_REGISTRY_URL` |
| `rover.apollo.dev` | Plugin and binary downloads. `rover dev` and `rover supergraph compose` download the composition plugin and router binary on first run. | When using `dev` or `supergraph compose` | `APOLLO_ROVER_DOWNLOAD_HOST` |
| `rover.apollo.dev` | Anonymous usage telemetry | No | Set `APOLLO_TELEMETRY_DISABLED=true` to disable. Set `APOLLO_TELEMETRY_URL` to redirect to an internal endpoint. |
| `rover.apollo.dev` | Template metadata for `rover template list` | No | `APOLLO_TEMPLATES_API` |
| `github.com` / `api.github.com` | Automatic version update checks that run in the background with every command. Template archive downloads for `rover template use` and `rover init`. | No | Set `APOLLO_ROVER_SKIP_UPDATE=true` to disable update checks. |
| `codeload.github.com` | GitHub's download CDN. `api.github.com` redirects template archive requests here when `rover template use` or `rover init` downloads a template. | When using those commands | — |
| `specs.apollo.dev` | Federation spec URLs embedded in schema directives. Rover does not fetch these at runtime—they are identifiers in your SDL. | No | — |

<Note>

Only `api.apollographql.com` is required for core registry operations. You only need the others when running specific commands.

</Note>

## Plugin downloads

`rover dev` and `rover supergraph compose` automatically download binaries from `rover.apollo.dev` on first run. In air-gapped or restricted environments, these downloads might fail without a clear error message.

<Caution>

If Rover cannot reach `rover.apollo.dev`, commands that require the composition plugin or router binary might fail. Download the required plugins on a machine with internet access and transfer them to the restricted environment, or include `rover.apollo.dev` in your allowlist.

</Caution>

## Environment variable reference

The following environment variables enable you to control which endpoints Rover contacts:

| Environment variable | Effect |
|---|---|
| `APOLLO_REGISTRY_URL` | Override the GraphOS API endpoint (default: `https://api.apollographql.com/graphql`) |
| `APOLLO_ROVER_DOWNLOAD_HOST` | Override the host used for plugin and binary downloads (default: `https://rover.apollo.dev`) |
| `APOLLO_TELEMETRY_URL` | Override the telemetry reporting endpoint (default: `https://rover.apollo.dev/telemetry`) |
| `APOLLO_TELEMETRY_DISABLED` | Set to `true` to disable telemetry entirely |
| `APOLLO_TEMPLATES_API` | Override the templates metadata endpoint (default: `https://rover.apollo.dev/templates`) |
| `APOLLO_ROVER_SKIP_UPDATE` | Set to `true` to disable automatic version update checks |

For all Rover environment variables, see [Rover configuration](./configuring).
